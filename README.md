# Создание MVP AI-ассистента для встреч (на базе Recall.ai)

Использование Bot-as-a-Service (BaaS) сервисов, таких как [Recall.ai](https://www.recall.ai/), — это стандартный паттерн для ИИ-ассистентов встреч, который снимает с вас всю инфраструктурную боль по поддержанию серверов, борьбе с обновлениями Zoom и обходу капч. В этой инструкции мы разберем создание MVP-версии бота по имени «Валера» для системы Sensum.

> **⚠️ ВАЖНО: Требования к аккаунту Recall.ai**  (и возможно ngrok)
> Сервис Recall.ai блокирует регистрации с российских доменов (например, `@yandex.ru`, `@mail.ru`, домены в зоне `.ru`). Для успешной регистрации и работы бота используйте почты международных сервисов, например **`@gmail.com`**.

---

## Этап 1. Подготовка локальной среды (Mac)

Поскольку разработка ведется на ноутбуке (MVP), API Recall.ai не сможет отправлять вебхуки напрямую на `localhost` или `127.0.0.1`. Для этого нужно "пробросить" локальный сервер в интернет с помощью утилиты **ngrok**.

## 1. Установка Homebrew

Для быстрой установки ngrok на Mac мы используем пакетный менеджер Homebrew. Если он еще не установлен, откройте Терминал и выполните команду:

`/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"`

_(После установки терминал может предложить выполнить пару команд в разделе "Next steps" для добавления brew в PATH — обязательно выполните их)._

## 2. Установка и запуск ngrok

После установки Homebrew выполните следующие шаги:

1. Установите ngrok командой:
    
    `brew install ngrok`
    
2. Зарегистрируйтесь на [ngrok.com](https://ngrok.com/) и получите свой Auth Token.
    
3. Авторизуйте ngrok в терминале:
    
    `ngrok config add-authtoken ВАШ_ТОКЕН`
    
4. Запустите туннель для порта 8000 (в первой вкладке Терминала):
    
    `ngrok http 8000`
    
    _(Если вы бронировали статический домен на ngrok, команда будет: `ngrok http --domain=ваш-домен.ngrok-free.app 8000`)_.
    
5. ngrok выдаст публичный URL (например, `https://abc12345.ngrok-free.app`). **Скопируйте его**, он понадобится в коде. Не закрывайте это окно терминала.

## Этап 2. Создание бэкенд-сервера (FastAPI)

Сервер Sensum будет состоять из одного файла, который выполняет три задачи: отправка бота в Zoom, вызов бота из Zoom и и удаление бота.

## 1. Установка библиотек

Откройте **вторую вкладку** Терминала, скачайте этот репозиторий и перейдите в его папку: 

`git clone https://github.com/Magasha/sensum-mvp-bot.git`
`cd sensum-mvp-bot`

Создайте виртуальное окружение и установите зависимости: 

`python3 -m venv .venv source .venv/bin/activate`
`pip install fastapi uvicorn requests`

## 2. Исправление кода (`main.py`)

**💡 Настройка кода перед запуском:**  
Обязательно проверьте и при необходимости замените:

- `zoom_link` — на актуальную ссылку Zoom (формат `https://zoom.us/j/...` или `https://us04web.zoom.us/j/...`).

- Значение `"url"` в блоке `real_time_endpoints` — на URL, который вам выдал ngrok на предыдущем шаге + `/webhook/recall`.

- `recall_api_key` — на ваш токен из дашборда Recall.ai.

---

## Этап 3. Тестирование MVP (Запуск бота)

Для проверки работы всего механизма необходимо выполнить следующие шаги.

1. **Запуск FastAPI сервера**  
    Во второй вкладке Терминала (убедившись, что ngrok работает в первой) выполните команду:
    
    `python3 -m uvicorn main:app --port 8000 --reload`
    
2. **Запуск Zoom**  
    Откройте десктопное приложение Zoom и начните новую встречу. Убедитесь, что ссылка совпадает с той, что указана в переменной `zoom_link` в `main.py`.
    
3. **Отправка команды**
    
    - Откройте браузер по адресу `http://127.0.0.1:8000/docs` (Swagger UI).
        
    - Раскройте метод `POST /api/trigger-bot`.
        
    - Нажмите **Try it out**, затем **Execute**.
        
    - В терминале бэкенда появится сообщение: `👉 Подключаем бота к...` и `✅ Успешно! ID бота: ...`.
        
4. **Работа в Zoom**
    
    - Бот постучится к вам в комнату ожидания (Waiting Room). Впустите его.
        
5. **Как отключить бота и завершить запись**  
    Чтобы принудительно выгнать бота из конференции (и запустить процесс формирования итоговой транскрипции на сервере Recall): 
    1. Скопируйте `bot_id`, который сервер выдал вам при успешном старте бота (в формате `7a4b9c2d-...`). 
    2. В интерфейсе Swagger (`http://127.0.0.1:8000/docs`) раскройте метод `POST /api/leave-bot`. 
    3. Нажмите **Try it out**, вставьте скопированный ID в поле `bot_id` и нажмите **Execute**. 
    4. Валера моментально выйдет из конференции Zoom, а в терминале появится сообщение `👋 Бот покинул встречу!`.

*Примечание: в текущей MVP-версии транскрипция собирается асинхронно. Итоговый текст звонка будет доступен для скачивания в личном кабинете Recall.ai или по API после того, как бот покинет встречу.*
